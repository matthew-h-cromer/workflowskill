# LinkedIn Job Search

```workflow
inputs:
  keywords:
    type: string
    default: "software engineer"
  location:
    type: string
    default: "United States"
  start:
    type: int
    default: 0

outputs:
  jobs:
    type: array
    value: $steps.extract_jobs.output.items

steps:
  - id: fetch_html
    type: tool
    tool: http.request
    description: Fetch raw HTML job listings from LinkedIn guest jobs API
    retry:
      max: 3
      delay: "2s"
      backoff: 2.0
    inputs:
      url:
        type: string
        value: "https://www.linkedin.com/jobs-guest/jobs/api/seeMoreJobPostings/search"
      method:
        type: string
        value: "GET"
      headers:
        type: object
        value:
          User-Agent: "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36"
          Accept: "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8"
          Accept-Language: "en-US,en;q=0.5"
    outputs:
      body:
        type: string
        value: $result.body

  - id: guard_empty
    type: exit
    description: Exit early if the request returned nothing
    condition: $steps.fetch_html.output.body == ""
    status: success
    output:
      jobs: []

  - id: extract_jobs
    type: tool
    tool: html.select
    description: Extract structured job posting fields from each <li> card
    inputs:
      html:
        type: string
        value: $steps.fetch_html.output.body
      selector:
        type: string
        value: "li"
      fields:
        type: object
        value:
          title: "h3.base-search-card__title"
          company: "h4.base-search-card__subtitle"
          location: "span.job-search-card__location"
          posted_at: "time"
          url: "a.base-card__full-link @href"
          company_url: "h4 a @href"
    outputs:
      items:
        type: array
        value: $result.results

  - id: guard_no_jobs
    type: exit
    description: Exit early if no job cards were found in the HTML
    condition: $steps.extract_jobs.output.items.length == 0
    status: success
    output:
      jobs: []
```

**A few things to be aware of:**

1. **Query parameters** — The workflow hits the base URL as-is. The `keywords`, `location`, and `start` inputs are declared but you'll need your runtime to support appending them as query params to the URL (e.g. `?keywords=...&location=...&start=...`), or you can concatenate them directly into the `url` value string if your expression language supports string interpolation.

2. **Bot blocking** — LinkedIn aggressively blocks scrapers. The `User-Agent` header helps, but at scale you'll likely need a rotating proxy service. The `retry` policy (3 attempts, exponential backoff) provides some resilience against transient blocks.

3. **Pagination** — Use the `start` input to paginate (e.g. `0`, `25`, `50`, …). Each page returns up to 25 job listings.
